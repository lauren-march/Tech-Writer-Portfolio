Use the `POST` method to add a new `/availability` resource.

## Create an availability record

Create a new availability record for a coliving or coworking space.

### Endpoint

```shell
POST /availability
```

### Request body

```json
{
  "spaceId": number,
  "availableSpots": number,
  "date": "string"
}
```

#### Example

```json
{
  "spaceId": 1,
  "availableSpots": 7,
  "date": "2025-12-11"
}
```

### Parameters

None.

### Responses

| HTTP Code | Description | Schema |
|-----------|-------------|--------|
| 201 | Availability record created successfully | `availability` object |
| 404 | Resource not found | `error` object |
| 400 | Bad request - invalid data | `error` object |

### Example request

#### cURL

```bash
curl -X POST http://localhost:3000/availability \
  -H "Content-Type: application/json" \
  -d '{
    "spaceId": 1,
    "availableSpots": 7,
    "date": "2025-12-11"
  }'
```

#### Postman

1. Set method to `POST`
1. Enter URL: `http://localhost:3000/availability`
1. Go to **Body** tab
1. Select **raw** and choose **JSON** from dropdown
1. Paste the request body JSON:

    ```json
        {
        "spaceId": 1,
        "availableSpots": 7,
        "date": "2025-12-11"
        }
    ```

1. Click **Send**

    ![](../media/post-availability-postman-example.png)

### Example response

```json
{
  "id": 5,
  "spaceId": 1,
  "availableSpots": 7,
  "date": "2025-12-11"
}
```

### Notes

* The `id` field is automatically generated by json-server
* All fields except `id` should be included in the request body
* Date must be in "YYYY-MM-DD" format
* `spaceId` must reference an existing space
* `availableSpots` should not exceed the space's capacity
